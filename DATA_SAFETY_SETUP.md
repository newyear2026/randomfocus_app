# Google Play Console - 데이터 수집 및 보안 설정 가이드

## 1. 필수 사용자 데이터 유형 수집 여부

### 질문: "앱에서 필수 사용자 데이터 유형을 수집하거나 공유하나요?"

**답변: 예**

**이유:**
- Google Mobile Ads를 사용하므로 **광고 ID**를 수집하고 Google과 공유합니다.
- 이는 "기기 ID 및 기타 식별자" 카테고리에 해당합니다.

---

## 2. 수집하는 데이터 유형 설정

Play Console에서 다음 데이터 유형들을 추가하고 설정해야 합니다:

### 📋 데이터 유형 1: 기기 ID 및 기타 식별자

#### 기본 정보:
- **데이터 유형**: 기기 ID 및 기타 식별자 (Device ID and other identifiers)
- **수집 여부**: 예
- **공유 여부**: 예

#### 상세 설정:

**1. 수집 목적:**
- ✅ **광고 또는 마케팅** 선택
  - 설명: "Google Mobile Ads를 통한 광고 표시를 위해 광고 ID를 수집합니다."

**2. 데이터 카테고리:**
- ✅ **기기 ID 또는 기타 식별자** 선택
  - 설명: "광고 ID (Advertising ID)를 수집합니다."

**3. 데이터 수집 여부:**
- ✅ **예, 이 데이터 유형을 수집합니다** 선택

**4. 데이터 공유 여부:**
- ✅ **예, 이 데이터 유형을 공유합니다** 선택
  - 공유 대상: Google (Google Mobile Ads)

**5. 데이터 보안 조치:**
- ✅ **전송 중 암호화** 선택
  - 설명: "HTTPS를 통해 암호화된 연결로 데이터를 전송합니다."

**6. 데이터 삭제:**
- ✅ **사용자가 요청하면 삭제** 선택
  - 설명: "사용자는 기기 설정에서 광고 ID를 재설정할 수 있습니다."

**7. 필수 여부:**
- ✅ **예, 이 데이터는 앱의 핵심 기능에 필수입니다** 선택
  - 설명: "광고 표시를 위해 필수입니다."

---

### 📋 데이터 유형 2: 앱 활동 (선택사항)

**참고**: 로컬 저장소에만 저장되는 데이터는 일반적으로 "필수 사용자 데이터"에 포함되지 않을 수 있습니다. 하지만 명확성을 위해 추가하는 것을 권장합니다.

#### 기본 정보:
- **데이터 유형**: 앱 활동 (App activity)
- **수집 여부**: 예 (로컬 저장소에만 저장)
- **공유 여부**: 아니오

#### 상세 설정:

**1. 수집 목적:**
- ✅ **앱 기능** 선택
  - 설명: "타이머 기록, 설정, 언어 선택 등 앱 기능 제공을 위해 로컬 저장소에 저장합니다."

**2. 데이터 카테고리:**
- ✅ **앱 상호작용** 선택
  - 설명: "타이머 세션 기록, 스핀 사용 기록, 앱 설정"

**3. 데이터 수집 여부:**
- ✅ **예, 이 데이터 유형을 수집합니다** 선택

**4. 데이터 공유 여부:**
- ❌ **아니요, 이 데이터 유형을 공유하지 않습니다** 선택
  - 설명: "모든 데이터는 기기 내부 저장소에만 보관되며 서버로 전송되지 않습니다."

**5. 데이터 보안 조치:**
- ✅ **로컬 저장소에 저장** 선택
  - 설명: "기기의 기본 보안 기능으로 보호되는 로컬 저장소에만 저장됩니다."

**6. 데이터 삭제:**
- ✅ **사용자가 요청하면 삭제** 선택
  - 설명: "앱 내 설정에서 기록을 삭제하거나 앱을 삭제하면 모든 데이터가 삭제됩니다."

**7. 필수 여부:**
- ✅ **예, 이 데이터는 앱의 핵심 기능에 필수입니다** 선택
  - 설명: "타이머 기록 및 설정 저장 기능을 위해 필수입니다."

## 3. 추가 배지 설정

### 3.1 독립적인 보안 검토를 거침

**이유:**
- 금융 앱이 아닙니다.

## 4. 단계별 설정 방법

### Step 1: 데이터 수집 여부 선택
1. "앱에서 필수 사용자 데이터 유형을 수집하거나 공유하나요?" 질문에
2. **"예"** 선택

### Step 2: 데이터 유형 추가
1. **"+ 데이터 유형 추가"** 버튼 클릭
2. **"기기 ID 및 기타 식별자"** 선택
3. 위의 상세 설정 내용 입력

### Step 3: (선택사항) 앱 활동 추가
1. **"+ 데이터 유형 추가"** 버튼 클릭
2. **"앱 활동"** 선택
3. 위의 상세 설정 내용 입력

### Step 4: 검토 및 제출
1. 모든 정보를 검토
2. **"저장"** 또는 **"다음"** 클릭

### 8.1 데이터 암호화 전송

## 9. 추가 질문 답변 (계속)

### 9.1 외부 계정 로그인

**질문**: "사용자가 앱 외부에서 만든 계정으로 앱에 로그인할 수 있나요?"

**답변: 아니요**

**이유:**
- 앱에 계정 생성 기능이 없습니다
- 로그인 기능이 없습니다
- 외부 계정(Google, Facebook 등)으로 로그인하는 기능이 없습니다
- 사용자 인증이 필요하지 않습니다

---

### 9.2 데이터 삭제 요청 방편

**질문**: "사용자가 데이터 삭제를 요청할 수 있는 방편을 제공하나요?"

**답변: 예**

**이유:**
1. **앱 삭제를 통한 데이터 삭제**
   - 앱을 삭제하면 모든 로컬 저장소 데이터가 자동으로 삭제됩니다
   - SharedPreferences에 저장된 모든 데이터(타이머 기록, 설정 등)가 삭제됩니다

2. **앱 내 삭제 기능 (코드 레벨)**
   - `HistoryService`에 다음 삭제 기능이 구현되어 있습니다:
     - `deleteHistory()` - 개별 기록 삭제
     - `deleteHistories()` - 여러 기록 삭제
     - `clearAllHistory()` - 모든 기록 삭제
   - 참고: 현재 UI에는 삭제 버튼이 표시되지 않지만, 기능은 구현되어 있습니다

3. **사용자 제어**
   - 사용자는 언제든지 앱을 삭제하여 모든 데이터를 삭제할 수 있습니다
   - 로컬 저장소에만 저장되므로 앱 삭제 시 완전히 제거됩니다

**설명 (Play Console에 입력할 내용):**
```
사용자는 앱을 삭제하여 모든 데이터를 삭제할 수 있습니다.
모든 데이터는 기기 내부 저장소에만 보관되므로 앱 삭제 시 
모든 타이머 기록, 설정, 언어 선택 등이 완전히 제거됩니다.
```
